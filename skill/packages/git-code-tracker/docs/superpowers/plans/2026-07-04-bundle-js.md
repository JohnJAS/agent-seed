# 打包 src/ 为单个 bundle.js — 实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking. Source spec: `docs/superpowers/specs/2026-07-04-bundle-js-design.md`.

**Goal:** 用 esbuild 将 `src/` 打包成单个 ESM `bundle.js`，部署到 `.opencode` 与 `.claude` 两个技能目录的 `scripts/` 下；两侧共 11 个 wrapper（`.opencode` 6 个 + `.claude` 5 个）改为从 `./bundle.js` 导入；删除散落的 `lib/` 目录树；升级流程同步清理旧 `lib/` 残留并复制 `bundle.js`。

**Architecture:** 新增 barrel `src/index.js` 汇总 7 个公开符号；新增 `scripts/build.js`（仓库根开发工具）单次 `esbuild.build` 写入两个目标；wrapper 仅改导入路径；`src/tracker/updater.js` 的 `downloadAndUpgrade` 抽出可测的 `applyReleaseFiles` 辅助函数，去除 `src→lib` 复制、`scriptsToCopy` 增加 `bundle.js`、两侧均 `fs.rm(lib)`。测试直接导入 `src/`，不依赖 bundle（`test/bundle.test.js` 例外，校验构建产物导出）。

**Tech Stack:** Node.js (ESM), esbuild (devDependency, 仅开发期), `node --test`

**约定:** 每个任务 = 一次原子提交。提交会自动触发 ai-code-tracker 钩子并生成配对的 `[ai-tracking]` 提交，属正常现象，无需干预。开始代码改动前先运行 ai-code-tracker 预检（AGENTS.md 强制要求）。

- [ ] **Step 3: 创建 `scripts/build.js`**

```javascript
import esbuild from "esbuild";
import fs from "node:fs/promises";
import path from "node:path";

const targets = [
  ".opencode/skills/ai-code-tracker/scripts/bundle.js",
  ".claude/skills/ai-code-tracker/scripts/bundle.js",
];

const repoRoot = path.resolve(import.meta.dirname, "..");

const result = await esbuild.build({
  entryPoints: [path.join(repoRoot, "src/index.js")],
  bundle: true,
  format: "esm",
  platform: "node",
  target: "node20",
  write: false,
  minify: false,
});

const code = result.outputFiles[0].text;
for (const target of targets) {
  const dest = path.join(repoRoot, target);
  await fs.mkdir(path.dirname(dest), { recursive: true });
  await fs.writeFile(dest, code, "utf8");
}
console.log(`bundle written to ${targets.length} targets`);
```

- [ ] **Step 2: 重构 `src/tracker/updater.js`**

新增导出函数 `applyReleaseFiles(repoRoot, extractDir)`，封装原 139–174 行逻辑并做三处改动：(1) 删除 `src→lib` 的 `fs.cp`；(2) `scriptsToCopy` 数组增加 `"bundle.js"`；(3) 在 `.claude` 同步前对两侧 `lib/` 执行 `fs.rm(..., { recursive: true, force: true })`。`downloadAndUpgrade` 在提取 tarball 后改为 `await applyReleaseFiles(repoRoot, extractDir)`。

```javascript
export async function applyReleaseFiles(repoRoot, extractDir) {
  const skillDest = path.join(repoRoot, SKILL_DIR);
  const srcSkillDir = path.join(extractDir, ".opencode", "skills", "ai-code-tracker");

  const scriptsSrc = path.join(srcSkillDir, "scripts");
  const scriptsDest = path.join(skillDest, "scripts");
  const scriptsToCopy = ["ai-update.js", "install.js", "commit-stats.js", "claude-code-hook.js", "ai-code-stats.js", "opencode-plugin.js", "bundle.js"];
  for (const script of scriptsToCopy) {
    try {
      await fs.copyFile(path.join(scriptsSrc, script), path.join(scriptsDest, script));
    } catch (error) {
      await logInfo(repoRoot, "updater.upgrade", `script ${script} not found in release`, { error: error.message });
    }
  }

  const commandsSrc = path.join(srcSkillDir, "commands");
  const commandsDest = path.join(skillDest, "commands");
  await fs.cp(commandsSrc, commandsDest, { recursive: true, force: true });

  try {
    await fs.copyFile(path.join(srcSkillDir, "SKILL.md"), path.join(skillDest, "SKILL.md"));
  } catch {}

  // 清理 .opencode 侧旧 lib/ 残留（已由 bundle.js 取代）
  await fs.rm(path.join(skillDest, "lib"), { recursive: true, force: true });

  // 同步 .claude 侧，并清理其旧 lib/ 残留
  const claudeSkillDest = path.join(repoRoot, CLAUDE_SKILL_DIR);
  try {
    await fs.stat(claudeSkillDest);
    await fs.rm(path.join(claudeSkillDest, "lib"), { recursive: true, force: true });
    await fs.cp(skillDest, claudeSkillDest, { recursive: true, force: true });
    await logInfo(repoRoot, "updater.upgrade", "synced .claude skill dir");
  } catch {
    // .claude skill dir may not exist; skip silently
  }
}
```

`downloadAndUpgrade` 中删除原 140–144 行（`srcSkillDir` / `srcLib` / `libDest` / `fs.cp(srcLib, libDest)`；其中 `srcSkillDir` 现由辅助函数内部重算，故一并删除）与原 146–174 行内联块，替换为：

```javascript
await applyReleaseFiles(repoRoot, extractDir);
```

`skillDest`（139 行）保留——后续 180 行运行 `install.js` 仍需使用。

- [ ] **Step 4: 冒烟验证（不执行 wrapper 顶层逻辑）**

  > **注意：** 5 个 import 型 wrapper（`commit-stats`/`ai-code-stats`/`ai-update`/`install`/`claude-code-hook`）在模块顶层即调用 `runX(...)`。动态 `import()` 会实际执行工具并对当前仓库产生副作用（真实 git 操作、写文件、出错时 `process.exit(1)`）。因此**不得**对 wrapper 做动态导入；改用以下三项安全检查：语法解析、导入路径文本、`bundle.js` 可加载性。

  ```bash
  # 1. 语法检查（node --check 只解析不执行，不触发 runX）
  for f in commit-stats ai-code-stats ai-update install claude-code-hook opencode-plugin; do
    node --check ".opencode/skills/ai-code-tracker/scripts/$f.js" || exit 1
  done
  for f in commit-stats ai-code-stats ai-update install claude-code-hook; do
    node --check ".claude/skills/ai-code-tracker/scripts/$f.js" || exit 1
  done

  # 2. 导入路径文本核对：11 个 wrapper 均含 from "./bundle.js"
  test "$(grep -rl '\./bundle\.js' .opencode/skills/ai-code-tracker/scripts .claude/skills/ai-code-tracker/scripts | wc -l)" = 11

  # 3. 两侧 bundle.js 可加载并导出所需符号（wrapper 的 ./bundle.js 相对路径解析验证）
  node -e "import('./.opencode/skills/ai-code-tracker/scripts/bundle.js').then(m=>{for(const n of ['runCommitStats','runAiCodeStats','runAiCodeUpdate','runInstall','runClaudeCodeHook','AiCodeTrackerPlugin','recordEditedFile'])if(!(n in m))throw new Error('missing '+n)}).catch(e=>{console.error(e);process.exit(1)})"
  node -e "import('./.claude/skills/ai-code-tracker/scripts/bundle.js').then(m=>{for(const n of ['runCommitStats','runAiCodeStats','runAiCodeUpdate','runInstall','runClaudeCodeHook'])if(!(n in m))throw new Error('missing '+n)}).catch(e=>{console.error(e);process.exit(1)})"
  ```

  三项均通过（`node --check` 无输出退出码 0；`test` 静默通过；两个 `node -e` 无 stderr）。`bundle.js` 本身无顶层副作用（barrel 仅 re-export，src 模块顶层只定义函数/常量，无 I/O），故直接导入是安全的。
